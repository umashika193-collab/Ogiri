# 大喜利トレーニング審査基準リサーチ

- 調査日: 2026-07-17
- 対象: アプリ内の10視点
- 目的: 実在人物の人格再現ではなく、公開情報を根拠にしたトレーニング用評価モデルの設計
- 実装状態: 12軸モデル、共通60%・人物別40%、確信度による共通基準への縮約を実装済み

## 結論

現在の共通軸である「お題への適合」「明瞭さ」「簡潔さ」「意外性」は研究上も妥当である。一方、10人の差を現在のような少数キーワードだけで表す根拠は不十分である。人物別モデルは、根拠の強さに応じて共通モデルへ寄せる必要がある。

最も根拠が強いのは、具体的な発想法や写真回答の作り方が本人の言葉で公開されている寺田◯明、バカリ◯ム、赤嶺総◯、粗◯。松本◯志は、IPPONの逐語記録、M-1公式記録、現在の大喜利大会で判断コメントを追加確認できたため低確信度から中確信度へ上げた。有吉◯行も、本人が判定基準と失敗への態度を語った資料、回答者を伏せた大喜利講評の実施記録を追加確認したため確信度を0.65から0.75へ上げた。ただし、IPPONグランプリの公式ルールでは出場者同士が採点するため、同番組の得点を本人固有の判断データとして扱うことはできない。

## 調査方法と証拠区分

- A: 本人が発想法・判断基準を直接説明したインタビュー
- B: 本人の具体的な審査コメントまたは制作過程の説明
- C: 公式番組ルール、公式紹介、受賞記録
- D: 信頼できる媒体による分析。人物別重みの主根拠にはせず、補助情報に限定

プロフィールの確信度は、A/B資料の量、発言の具体性、大喜利への直接性で決めた。漫才・コントの発言を大喜利へ移す場合は一段階弱い証拠として扱う。

## 共通評価軸

| ID | 評価軸 | 操作的な定義 |
|---|---|---|
| promptFit | お題適合 | お題の対象・質問形式・写真を踏まえて回答している |
| clarity | 明瞭さ | 一度で状況と笑いどころが理解できる |
| brevity | 簡潔さ | 必要な情報を残し、説明過多になっていない |
| novelty | 新鮮さ | 既視感の強い型や先行回答と重複しない |
| surprise | 意外性 | 聞き手の最初の予測から離れている |
| twist | ずらし | お題との関係を保ったまま別角度へ移動している |
| specificity | 具体性 | 固有の人物像、場所、数字、役割、物語の手掛かりがある |
| escalation | 展開 | 一段強い着地、かぶせ、勾配、記憶に残る瞬間がある |
| rhythm | 言葉の着地 | 語順、間、音、最後の一語が機能している |
| visualGrounding | 写真接続 | 写真の物体、形、色、関係、異常点を利用している |
| instant | 瞬発性 | 短時間で回答している。ただし面白さ本体とは分離する |
| edge | 切れ味 | 判断や視点を曖昧にせず、回答が言い切られている |

大喜利研究では、お題を踏まえること、分かりやすさ、簡潔さ、現実との距離、想定との差、新鮮さが評価項目として使われている [S01]。このため、`promptFit`、`clarity`、`brevity`、`novelty`、`surprise` は全プロフィールに最低限残す。

## 人物別仮説

### 松本◯志 — 発想の飛距離、型の切替、回答列の展開

- 仮説: `novelty`、`surprise`、`twist` を中心に、`promptFit`、`clarity`、`escalation` を組み合わせる。
- 大喜利への直接根拠: IPPONの放送逐語記録では、本人が難しいお題の考え方を「モデルとなる対象をどこまで意識するか」と「隠し事の種類をどれだけ考えるか」の二方向に分解している [S23]。回答へのコメントでは、新しい登場人物を足す視点、絵で答える型、本人らしくない型への切替、一発の爆発力、回答列の途中で型を崩して遊ぶ展開を個別に識別している [S23]。
- 制作上の根拠: IPPONでは多数の候補から本人も参加してお題を選び、チェアマンでありながら自分でも回答を作ると番組作家が説明している [S27]。したがって、完成回答だけでなく、お題の答えやすさと回答空間も判断対象に含むと推定する。
- 他形式からの補助根拠: M-1公式記録では「意外と新しい試み」「新しいところ」、伝統的形式の完成度、単純明快なのに笑えることをそれぞれ評価している [S24][S25][S26]。これは新規性一辺倒ではなく、既存形式でも明快さと完成度が高ければ評価することを示す。ただし漫才審査なので補助証拠に限定する。
- 現在の活動による補強: 2026年の大喜利大会では、本人と出題者が共同で採点し、回答者へ「攻めても大丈夫」と方向を示しながら、その場で回答ごとの差をコメントしている [S28]。別企画では即興、3日後、1週間後の回答を比較しており、回答時間と面白さを同一視せず検証対象として分けている [S29]。
- 注意: IPPONの得点は本人の投票ではない。公開資料から本人の1〜10点の境界値や、同じ回答に対する再現性は復元できない。人物の完全再現ではなく「新規性・転換・展開を重く見る訓練視点」として使う。
- 確信度: 0.65（中。追加調査前0.35）
- 詳細: [`matsumoto-deep-dive.md`](matsumoto-deep-dive.md)

### 有吉◯行 — 主観の即時反応、人物文脈、失敗許容

- 仮説: `edge`、`clarity`、`brevity` を土台に、`promptFit`、`escalation`、`visualGrounding`を組み合わせる。速度は補助軸に下げる。
- 本人発言: 公開インタビューでは、全国的な成功可能性ではなく自分が単純に面白いと思うかで判断し、自分の判断と世間の評価がずれる可能性も認めている [S06][S07]。したがって「一般受けの予測」ではなく、その場で笑いの焦点が明確かを中心にする。
- 人物文脈: 『有吉の壁』の○×について、基本的には笑いやすく、ネタ単体より演者の人物像や、安心して失敗を見られる関係まで含める趣旨を本人が説明している [S30]。同じ取材では、下ネタの境界を共演者の実際の反応で調整する趣旨も語っており、場と受け手の反応を無視しない判断が見える。
- 大喜利への直接性: 『大喜利ドリル』では、事前提出され作者名を伏せた回答へ講評する形式が確認できる [S32][S33]。人物への先入観を除いた回答単体の判断も行っていることは分かるが、全回答・全講評・採点値は公開されていない。
- 周辺証言: 他者を判定するだけでなく自分でも回答し、外した回答も処理して次へ進める点が説明されている [S05]。
- 注意: 「毒舌語」や攻撃性を高得点条件にしない。人物文脈は現行の一文採点だけでは観測できないため、現在は明快な視点、回答の言い切り、失敗後の再挑戦支援へ縮約する。
- 確信度: 0.75（中〜高。追加調査前0.65）
- 詳細: [`ariyoshi-deep-dive.md`](ariyoshi-deep-dive.md)

## 実装した設計原則

1. 総合点の60%を共通基準、40%を人物別視点にする。
2. 確信度が低い人物ほど、人物別重みを共通基準へ縮約する。
3. 表示は「本人の採点」ではなく「公開情報を参考にした10のトレーニング視点」とする。
4. `instant` は面白さそのものと分離し、練習ボーナスとして最大1点程度に抑える。
5. キーワードの有無だけで毒・意外性・ひねりを決めない。意味の距離、写真内要素、先行回答との重複を別々に評価する。
6. 写真で一言では、物体名の一致だけでなく、位置関係、用途の誤読、擬人化、写真外の物語を測る。
7. 更新は、出典、設定版、採点版、特徴量版、検証用回答セットを残す。

## 現在も残るモデル上の限界

- `novelty` と `surprise` は語彙カテゴリ、文字の多様性、合図語による決定的な近似であり、巨大な既出回答群との意味比較ではない。
- `visualGrounding` は画像の色・明暗ラベルと単語一致が中心で、物体・関係・構図の意味理解はない。
- `clarity` と `rhythm` は文字数、文末、句読点の比重が高い。
- 同じ回答文の既出度や、同じお題内の重複を評価していない。

## 人間評価の実装状態

- 安全な人工回答を各お題20件、10題、合計200件用意した。
- `validation.html` で共通12軸と総合的な面白さを匿名評価できる。
- `scripts/analyze-ratings.js` で評価者間一致とモデル点との順位相関を測定できる。
- 3人以上の実在評価者による採点そのものは外部協力が必要で、未収集である。AIによる架空の人間票で代用しない。
- 低一致のお題では人物差を出さず「判定困難」と表示する。
- 公開番組の回答全文を大量保存・再配布せず、必要なら特徴量と出典位置だけを記録する。
