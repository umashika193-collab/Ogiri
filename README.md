# ボケトレ！— 森羅万象・大喜利トレーニング

身近な物・風景・偶然の構図を教材にする、スマートフォン向け大喜利トレーニングPWAです。カメラ撮影では人物を対象外とし、著作権上安全なフリー画像では人物も教材にできます。声か文字で回答すると、APIキー不要の端末内分析だけで練習できます。

実在人物本人を再現する製品ではありません。画面上の伏字コーチは、発想・毒と短さ・構造と速度・観察・異物感・完成度・選択・情景・設定・言葉という独立した評価タイプです。

## 遊び方

1. 「今日のトレーニングを始める」を押す
2. 「自分で撮る」または「フリー素材」を選ぶ
3. どちらのモードでも、表示された写真に対して「この写真で一言。」へ答える
4. フリー素材では、4回ごとに同梱写真3点とWikimedia Commonsの実写JPEG 1点を順不同で使い、「別の素材にする」で回答前に交換できる
5. お題番長・一言カッター・写真探偵の三役審査と、研究データから選ばれた練習コース別アドバイスを確認する
6. 同じ写真へ再挑戦するか次の写真へ進む

## 端末内分析

通常プレイでは、端末内の決定的なルールでローカルトレーニング値を計算します。同じ写真特徴・回答・回答時間なら同じ結果になります。お題は全モードで「この写真で一言。」に固定しています。この値はユーモアの絶対評価ではなく、端末で測定できる形式上の練習指標です。

- お題形式への適合
- 明瞭さと短さ
- 具体性と言葉の着地
- 写真の色・明暗などの特徴語との接続
- 回答速度

結果画面では、同じ7信号を役割ごとにまとめた三役が、それぞれ短い講評を返します。講評は決定的なルールから生成され、外部AIへの送信やAPIキーを必要としません。

- お題番長: お題形式、明瞭さ、具体性
- 一言カッター: 短さ、言葉の着地、回答速度
- 写真探偵: 写真特徴、具体性、明瞭さ

三役は文章の意味やユーモアを理解するAIではありません。点数は測定可能な形式上の練習値で、面白さの絶対評価ではありません。

公開情報から作成した10のトレーニング視点は、得点へ加算せず、発想・観察・設定・言葉などの練習コースとして使用します。各回答の測定可能な7信号を検証用200回答の軸別平均と比較し、10視点の研究済み重みを照合してコースと具体的なアドバイスを選びます。同じコースが続かないようセッション内でローテーションし、「役立つ／微妙」の評価も次回の選択へ端末内で反映します。実在人物本人の採点や発言ではありません。

練習コースの選択には、新鮮さ・意外性・ひねりなど端末内で意味を検証できない軸を使いません。コース名、平均との差、選択根拠は結果画面で確認でき、三役によるローカルトレーニング値はコース設定を変更しても変わりません。

## プライバシーと安全性

- カメラとマイクは、使う操作をした時だけ許可を求めます。
- 位置情報、連絡先、写真ライブラリ、メッセージ、閲覧履歴にはアクセスしません。
- 写真、音声、回答文は永続保存しません。
- 保存するのはベスト点とヒント評価だけで、画面から削除できます。
- 採点と改善ヒントの生成に外部AIやAPIキーを使用しません。
- ブラウザの音声認識は、端末やブラウザの実装により外部処理される場合があります。
- オンライン時もフリー素材4回のうち3回は同梱写真、1回だけWikimedia Commonsを利用します。通信失敗時はすぐ同梱写真へ戻ります。
- Commons画像は自由利用できるライセンスが表示された実写JPEGに限定し、ロゴ、ポスター、地図、図表、広告、文書、絵画、イラストなどをメタデータで除外します。HTTPS・ホスト・MIME・容量・タイムアウトも検証します。
- Commons画像では原題を画面に出さず、作者・ライセンス・原典リンクだけをクレジット表示します。文字そのものは許可しますが、英文主体になりやすいポスターや文書類は除外します。
- カメラ撮影の人物検出は補助機能であり、完全ではありません。検出非対応時は利用者による確認を必須にしています。Wikimedia Commonsのフリー素材は人物を含む場合があります。
- 外部入力は `textContent` とDOM生成で表示し、CSPで通信先と読み込み元を制限しています。

## オフライン

Service Workerがアプリ本体と、現実世界の少し変な場面を写した45種類の実写風素材（AI生成）をキャッシュします。通信できない場合は、会議室の椅子、海辺の買い物カート、田んぼの事務椅子、バスを待つ牛、温室のピアノなどを一巡するまで重複させずに使います。更新途中などで一部の画像シートを読み込めない場合は、そのシートを一時的に除外して残りを探します。すべて読み込めない場合も意味のない図形は生成せず、再試行案内を表示します。文字回答と端末内採点はそのまま続行でき、音声認識のオフライン可否はブラウザに依存します。

PCで`index.html`を直接開く`file://`起動にも対応しています。この場合、ブラウザのローカルファイル制限により画像ピクセルの詳細解析だけを省略し、同梱した特徴語で採点を続けます。Service WorkerとWikimedia Commons取得を含む通常動作の確認にはローカルWebサーバーまたは公開URLを使用してください。

## インストール

埋め込みブラウザでは、タイトル画面に標準ブラウザで開く導線とURLコピーを表示します。ChromeやEdgeなどインストール対応ブラウザでは「アプリをインストール」を表示し、iOS Safariなど直接プロンプトを出せない環境では「ホーム画面に追加」の手順を案内します。インストール済みのPWAでは案内を表示しません。

## ローカル起動

静的サーバーだけで起動できます。

```powershell
python -m http.server 4173
```

`http://127.0.0.1:4173/` をスマートフォン相当のブラウザで開いてください。カメラなど一部機能はHTTPSまたはlocalhost系の安全なコンテキストが必要です。

12軸の重みデータは実行時に読み込まれます。通常プレイでは、そのうち端末で測定可能な7信号に対応する部分だけを練習コース選択へ使用し、得点には使用しません。公開資料にない人格・口調・私生活はモデル化しません。

## 匿名評価

`http://127.0.0.1:4173/validation.html` で、10題×20回答を12軸と総合点で匿名評価できます。3人以上の評価JSONを集めた後、次の形式で評価者間一致とモデル点との順位相関を確認します。

```powershell
node scripts/analyze-ratings.js ratings-a.json ratings-b.json ratings-c.json
```

評価者が3人未満の場合、解析結果は `needs-more-human-raters` となり、検証完了とは扱いません。

## テスト

```powershell
node --check scoring-core.js
node --check stock-library.js
node --check game.js
node --check service-worker.js
node --test tests/pwa-ui.test.js tests/scoring.test.js tests/stock-library.test.js
```

ローカル分析の再現性、画像特徴分類、三役審査が測定可能な信号だけを使うこと、旧10視点が得点へ影響しないこと、練習コースと素材の3:1ローテーション、Commonsの実写JPEG絞り込みを検証します。
