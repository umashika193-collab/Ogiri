# ボケトレ！— 森羅万象・大喜利トレーニング

人を除く身近な物・風景・偶然の構図を教材にする、スマートフォン向け大喜利トレーニングPWAです。カメラまたは著作権上安全なフリー画像を選び、声か文字で回答すると、10種類の評価軸から採点・改善ヒント・再挑戦時の伸び幅を確認できます。

実在人物本人を再現する製品ではありません。画面上の伏字コーチは、発想・毒と短さ・構造と速度・伝達・違和感・完成度・即効性・情景・設定・言葉という独立した評価タイプです。

## 遊び方

1. 「今日のトレーニングを始める」を押す
2. 人物を含まない対象を撮影するか、「フリー画像で遊ぶ」を選ぶ
3. 提示されたお題へ、声または文字で答える
4. 採点根拠と改善ヒントを確認する
5. 同じお題へ再挑戦するか、次の教材へ進む

## 採点

採点は端末内の決定的なルールで行います。同じ写真特徴・お題・回答・回答時間なら同じ結果になります。

- 写真の明暗、色、彩度、密度、構図
- 写真と回答の語彙上の接続
- お題の型への適合
- 意外性、ひねり、短さ、明瞭さ、リズム
- 回答速度、具体性、エッジ

点数はユーモアの絶対評価ではなく、反復練習の比較指標です。「役立つ／微妙」の評価は、次回の改善ヒント選びへ端末内で反映されます。

## プライバシーと安全性

- カメラとマイクは、使う操作をした時だけ許可を求めます。
- 位置情報、連絡先、写真ライブラリ、メッセージ、閲覧履歴にはアクセスしません。
- 写真、音声、回答文は永続保存しません。
- 保存するのはベスト点とヒント評価だけで、画面から削除できます。
- ブラウザの音声認識は、端末やブラウザの実装により外部処理される場合があります。
- フリー画像はOpenverseのCC0またはPublic Domain Markに限定し、HTTPS・ホスト・MIME・容量・タイムアウトを検証します。
- 人物検出は補助機能であり、完全ではありません。検出非対応時は利用者による確認を必須にしています。
- 外部入力は `textContent` とDOM生成で表示し、CSPで通信先と読み込み元を制限しています。

## オフライン

Service Workerがアプリ本体をキャッシュします。通信できない場合、教材画像を端末内で生成し、文字回答と端末内採点で続行できます。音声認識のオフライン可否はブラウザに依存します。

## ローカル起動

```powershell
python -m http.server 4173
```

`http://127.0.0.1:4173/` をスマートフォン相当のブラウザで開いてください。カメラなど一部機能はHTTPSまたはlocalhost系の安全なコンテキストが必要です。

## テスト

```powershell
node --check scoring-core.js
node --check game.js
node --check service-worker.js
node --test tests/scoring.test.js
```

採点の再現性、写真・お題による変化、ヒント評価の反映、画像特徴分類を検証します。
