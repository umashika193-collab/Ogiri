# ボケトレ！— 森羅万象・大喜利トレーニング

身近な物・風景・偶然の構図を教材にする、スマートフォン向け大喜利トレーニングPWAです。カメラ撮影では人物を対象外とし、著作権上安全なフリー画像では人物も教材にできます。声か文字で回答すると、APIキー不要の端末内分析だけで練習できます。

実在人物本人を再現する製品ではありません。画面上の伏字コーチは、発想・毒と短さ・構造と速度・観察・異物感・完成度・選択・情景・設定・言葉という独立した評価タイプです。

## 遊び方

1. 「今日のトレーニングを始める」を押す
2. 「自分で撮る・写真で一言」または「フリー素材・大喜利」を選ぶ
3. 写真で一言モードでは人物を含まない対象を撮影し、「この写真で一言。」へ答える
4. フリー素材大喜利では人物も含まれるOpenverse画像とランダムなお題へ答える
5. お題番長・一言カッター・写真探偵の三役審査と改善ヒントを確認する
6. 同じお題へ再挑戦するか次の教材へ進む

## 端末内分析

通常プレイでは、端末内の決定的なルールでローカルトレーニング値を計算します。同じ写真特徴・お題・回答・回答時間なら同じ結果になります。この値はユーモアの絶対評価ではなく、端末で測定できる形式上の練習指標です。

- お題形式への適合
- 明瞭さと短さ
- 具体性と言葉の着地
- 写真の色・明暗などの特徴語との接続
- 回答速度

結果画面では、同じ7信号を役割ごとにまとめた三役が、それぞれ短い講評を返します。講評は決定的なルールから生成され、外部AIへの送信やAPIキーを必要としません。

- お題番長: お題形式、明瞭さ、具体性
- 一言カッター: 短さ、言葉の着地、回答速度
- 写真探偵: 写真特徴、具体性、明瞭さ

三役は文章の意味やユーモアを理解するAIではありません。点数は測定可能な形式上の練習値で、面白さの絶対評価ではありません。

公開情報から作成した10のトレーニング視点は、改善ヒントと研究用設定として維持しています。実在人物本人の採点ではありません。「役立つ／微妙」の評価は次回のローカル改善ヒント選びへ端末内で反映されます。

## プライバシーと安全性

- カメラとマイクは、使う操作をした時だけ許可を求めます。
- 位置情報、連絡先、写真ライブラリ、メッセージ、閲覧履歴にはアクセスしません。
- 写真、音声、回答文は永続保存しません。
- 保存するのはベスト点とヒント評価だけで、画面から削除できます。
- 採点と改善ヒントの生成に外部AIやAPIキーを使用しません。
- ブラウザの音声認識は、端末やブラウザの実装により外部処理される場合があります。
- フリー画像はOpenverseのCC0またはPublic Domain Markに限定し、HTTPS・ホスト・MIME・容量・タイムアウトを検証します。
- カメラ撮影の人物検出は補助機能であり、完全ではありません。検出非対応時は利用者による確認を必須にしています。Openverseのフリー素材は人物を含む場合があります。
- 外部入力は `textContent` とDOM生成で表示し、CSPで通信先と読み込み元を制限しています。

## オフライン

Service Workerがアプリ本体をキャッシュします。通信できない場合、教材画像を端末内で生成し、文字回答と端末内採点で続行できます。音声認識のオフライン可否はブラウザに依存します。

## インストール

埋め込みブラウザでは、タイトル画面に標準ブラウザで開く導線とURLコピーを表示します。ChromeやEdgeなどインストール対応ブラウザでは「アプリをインストール」を表示し、iOS Safariなど直接プロンプトを出せない環境では「ホーム画面に追加」の手順を案内します。インストール済みのPWAでは案内を表示しません。

## ローカル起動

静的サーバーだけで起動できます。

```powershell
python -m http.server 4173
```

`http://127.0.0.1:4173/` をスマートフォン相当のブラウザで開いてください。カメラなど一部機能はHTTPSまたはlocalhost系の安全なコンテキストが必要です。

## 匿名評価

`http://127.0.0.1:4173/validation.html` で、10題×20回答を12軸と総合点で匿名評価できます。3人以上の評価JSONを集めた後、次の形式で評価者間一致とモデル点との順位相関を確認します。

```powershell
node scripts/analyze-ratings.js ratings-a.json ratings-b.json ratings-c.json
```

評価者が3人未満の場合、解析結果は `needs-more-human-raters` となり、検証完了とは扱いません。

## テスト

```powershell
node --check scoring-core.js
node --check game.js
node --check service-worker.js
node --test tests/pwa-ui.test.js tests/scoring.test.js
```

ローカル分析の再現性、画像特徴分類、三役審査が測定可能な信号だけを使うことを検証します。
