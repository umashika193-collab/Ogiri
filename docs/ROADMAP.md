# 開発ロードマップ

## Phase 0 — 実装基盤

- 現行プロトタイプを参照実装として凍結
- TypeScriptプロジェクト作成
- 状態機械導入
- 採点エンジン分離
- 自動テスト環境作成

完了基準：採点エンジンをDOMなしでテストできる。

## Phase 1 — Training MVP

- カメラ、音声、Openverse、生成画像
- 10軸100点分析
- 改善ヒント
- 同一問題への再挑戦
- 得点差
- 端末内履歴
- 音声処理と第三者送信可能性の明示
- プライバシーポリシー・削除・権限撤回UI
- 外部画像URL検証とコンテンツ安全対策

完了基準：スマートフォン実機で10回連続完走する。

## Phase 2 — 継続学習

- 連続練習日数
- 週間レポート
- 能力別推移
- お題難易度
- ヒント有効率
- 苦手軸に応じた出題

完了基準：ユーザーが自分の成長理由を説明できる。

## Phase 3 — PWA公開

- プライバシーポリシー
- 利用規約
- オフライン品質
- アクセシビリティ
- エラー監視
- 限定公開テスト

完了基準：限定ユーザーの7日継続率と再挑戦率を計測できる。

## Phase 4 — アプリ化判断

- PWAの利用データを評価
- iOS / Androidラッパーの必要性判断
- 通知、ホーム画面、バックアップ要件の確定
- ストア表示・審査要件の整理

判断指標：再挑戦率、7日継続率、週3回以上の利用率。
