# デバッガー・ホワイトハッカー合同レビュー

レビュー日：2026-07-17
対象：現行HTMLプロトタイプおよび実装準備資料
結論：P0なし。ただしP1を解消するまで公開・アプリ化へ進まない。

## P1 — リリースブロッカー

### 1. 採点の応答時間が非決定的

現状は審査演出のランダム待ち時間後に応答時間を計測するため、同じ発話でも点数が変わり得る。

対策：音声回答終了時点で `responseMs` を確定し、純粋な採点関数へ渡す。演出時間を採点入力に含めない。

### 2. 音声非対応・オフライン・開始例外で進行不能

音声認識非対応でも回答ボタンが表示され、押しても何も起きない。Web Speech APIは完全オフラインを保証できない。

対策：対応状況に基づく状態表示、開始・停止例外処理、復帰導線、代替入力方針を確定する。完全オフライン音声を要件とする場合は端末内STTを採用する。

### 3. 音声プライバシー表示が実装と一致しない

ブラウザ・OSの音声認識実装によって音声が外部処理される可能性があるが、現行表示は外部送信しないと断定している。

対策：録音開始前に処理主体、送信可能性、保存有無を説明する。クラウドSTTは明示的オプトインとする。

### 4. カメラ操作の競合と停止漏れ

映像準備前の撮影、カウントダウン中のカメラ切替、`drawImage` 例外で操作不能になり得る。撮影後もカメラが終了まで動作する。

対策：`loadedmetadata` / `playing` を待つ。撮影中は競合操作をロックする。失敗時は `finally` で復帰する。撮影完了、フリー画像選択、バックグラウンド移行で全trackを停止する。

### 5. 権限要求が過剰で一括

位置、方角、傾き、加速度、電池、カメラを開始時にまとめて扱っている。位置は丸める前に精密座標を取得する。

対策：位置情報はMVPから削除する。残る権限も機能を使う直前に個別要求する。センサーlistenerの解除と値の破棄を行う。

### 6. 人物除外を保証できない

FaceDetectorは非対応・失敗時に通過し、背面・横顔・身体を検出できない。Openverse画像は英語メタデータだけで除外している。

対策：人物全身を対象にしたオンデバイス検出、信頼度閾値、失敗時拒否を導入する。検出不能端末ではカメラ教材を無効化するか、保証不能を明示する。外部画像も内容検査済みの教材を優先する。

### 7. 画像と回答の関連性を評価していない

採点エンジンは写真とお題を入力としておらず、同じ回答なら異なる教材でもほぼ同点になる。

対策：関連性を評価しないMVPとして明記するか、安全な画像・意味解析を独立軸として追加する。関連性がない状態で「写真への適合」を表示しない。

### 8. Openverseと外部画像が未信頼境界

APIから得た画像URLと原典URLをスキーム・ホスト検証なしで使用する。外部画像ホストへIPやUAが伝わり、人物、NSFW、悪性誘導、誤ライセンスの余地がある。

対策：HTTPS、ホスト、MIME、サイズ、タイムアウトを検証する。外部リンクへ `noreferrer` を付ける。CSPを導入する。製品版では安全検査と短期キャッシュを行う画像プロキシを検討する。

### 9. 公開前のプライバシー統制が不足

プライバシーポリシー、第三者一覧、削除UI、権限撤回UI、年齢・コンテンツ方針が未実装。

対策：これらをPhase 3ではなくMVP公開前のリリースブロッカーへ移す。

### 10. 自動テストが存在しない

受け入れテストは未実施で、既存のテスト動画は旧仕様。採点、状態遷移、権限拒否、音声イベント競合、カメラ競合、再挑戦、オフラインを検証できない。

対策：採点純粋関数のユニットテスト、状態機械テスト、ブラウザ統合テスト、実機テストを導入する。

## P2 — MVP中に解消

### Service Worker

- ナビゲーション以外の失敗にもHTMLを返している。
- 非2xx応答もキャッシュし得る。
- same-origin GETを無制限にキャッシュする。

対策：シェルallowlist、navigationのみHTML fallback、`response.ok` とMIME検証、上限・期限、API・認証・履歴のno-storeを実装する。

### PWA

- manifestにアイコンがない。
- インストール要件を満たせない端末がある。

### 画面・アクセシビリティ

- `user-scalable=no` と `overflow:hidden` により、小型画面や文字拡大で操作が欠落し得る。
- ズームを禁止しない。開始画面と結果画面をスクロール可能にする。

### LocalStorage

- 破損JSON、無効化、容量制限で停止し得る。
- 全消去UIと保持期限がない。

対策：storage adapter、スキーマ検証、try/catch、IndexedDB、全消去、回答文保存のオプトインを実装する。

### DOM・ブラウザセキュリティ

- 現時点で外部入力由来のDOM XSSは確認されていない。
- `innerHTML` を減らし、DOM生成へ置き換える。
- CSP、Permissions-Policy、Referrer-Policy、X-Content-Type-Optionsを導入する。
- Google Fontsは自己配信する。

### コンテンツ安全性

- 人物、裸体、暴力、差別、個人情報の制御が不足する。
- 危険語を面白さへ直接加点しない。
- 年齢区分、利用ルール、画像再取得、通報導線を設ける。

## P3 — 製品ビルドから除外

- `?demo` によるカメラ・人物ガード迂回
- 旧仕様の `testplay.html`、動画、フレーム
- タイムアウトのない外部fetchと画像読み込み

## アプリ化時の追加ゲート

- WebViewの遷移・ディープリンクallowlist
- file access / universal accessの無効化
- 最小限のmanifest権限
- HTTPSのみ
- バックグラウンド時のカメラ・マイク停止
- アプリ切替画面での内容秘匿
- クラッシュ解析へ回答・位置・外部URLを送信しない
- 署名済み採点設定、バージョン管理、ロールバック
- App Store Privacy / Google Play Data Safetyの正確な申告
